Blokko是一个简易的置顶图生成器，目的是解决简体中文社区微博置顶/扩列图/一类的需求，目前已经支持了富文本、药丸标签、纹理、背景图、渐变、瀑布流展示、响应式布局等多种功能，并对手机端完成了基础适配，部署在GitHub Page和Cloudflare Page上。

---

国内用户可以尝试使用第一个链接，但是如果你能看到这段文字，那第二个大概率也是通的，首次加载或版本更新后会需要加载一点点资源，已知火狐浏览器会出现预览问题，建议使用Chrome/Edge等Chromium内核的浏览器，手机端目前仅推荐在启用全屏模式下关闭夜间模式使用。

https://baozicola.pages.dev/ （Cloudflare)

https://baozicola.github.io/Blokko/  （GitHub page）

---


Blokko采用三栏式布局，左侧为图层，中间为预览，右侧为样式（检查器），PC状态下，右侧会显示全局面板，在中间预览区域点击对应元素，右侧则会跳转到对应的卡片的独立设置界面，可以随时通过最顶上返回全局设置，界面上绝大多数元素都支持拖动，文本均支持双击修改文字。

而在手机端，基础操作不变，但图层和样式会变为左右折叠抽屉，需要手动点击顶部按钮呼出，为了手机体验，当长按不拖动某一卡片时，会自动呼出对应卡片的设置界面，手机端也可以额外点击铅笔图标在预览界面编辑文字（双击也可以，但是不灵敏），预览时也可以打开锁定模式上下滑动，以免触发意外拖动打乱布局。

QA部分

Q：为什么是建议使用Edge/Chrome/Via

A：因为我就在用这三个浏览器，而且基数大，还有就是window.queryLocalFonts(读取本地字体API)在电脑端Edge和Chrome可以使用，如果你当前使用的浏览器是Chromium内核，那大概也能正常用字体读取。

Q：首次打开没有中间的预览界面怎么办？

A：请先确认最上方"帮助"右侧是否有一个"?"图标，如果无，则说明是网络问题，尝试切换网络使用，因为引用了外部CDN，在网络较差的情况下打不开是正常情况。

Q：出现初始化失败怎么办。

A：先尝试在紧急弹窗里下载本地数据，然后再刷新浏览器，如果多次未果就点击重置数据，或者手动清理浏览器缓存，理论上通过刷新浏览器是可以直接解决的(只要我没有把测试版本发上来的话)。

Q：怎么改单张卡片的设置？

A：单个卡片的调节，需要预览区/左侧图层找到对应的卡片，点一下，右侧就会进入独立设置，可以进入独立设置对单个卡片修改。

Q：手机端图片上传无显示

A：这是因为浏览器性能原因，尽管Blokko已经做了图片压缩，但还是不可避免在一些设备上出现这种情况，建议更换图片或尝试修改为PC端

Q：图片导出出现图片区块的瀑布流重叠

A：修不好！我尽力了Orz，目前这个BUG会在完全不变动区块第一次导出图片，然后退出来再直接导出第二次图片的时候触发，我也排查不出来是哪里有问题，好消息是这个BUG只需要刷新页面/随便动一下区块元素就可以解决了，而且是第二次导出图片才会触发，一般不影响使用，准备以后再修了。

Q：出现初始化失败怎么办。

A：先尝试在紧急弹窗里下载本地数据，然后再刷新浏览器，如果多次未果就点击重置数据，或者手动清理浏览器缓存，理论上通过刷新浏览器是可以直接解决的(只要我没有把测试版本发上来的话)。

Q：四种导出数据方式的优缺点是什么？

A：标准导出速度快，导入快，但是不带图片，下次导入需要导入图片；强化导出略慢，导入也略慢，但可以完整备份创作数据，~~就算哪天删库跑路了你也能手动解压压缩包去别的软件用~~ ；模板导出只会备份颜色布局卡片，不备份图片和文字信息，初衷是分享配色方案等使用，导出速度导入都很快；过时导出是曾使用的方案，该方案集导出慢导入慢容易崩众多缺点为一体，不要用。

Q：为什么会有水印？可以取消水印吗？
A：水印只会在使用了Pixabay图像的时候被默认勾选并可以主动关闭，根据Pixabay协议，使用图片尽管可以不用署名，但我仍然鼓励大家勾选水印并分享背景图来源。如果自己上传图片，只要在导出面板里不主动勾选，就不会有水印。
